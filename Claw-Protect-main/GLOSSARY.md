# Glossary

> Canonical definitions for terms used throughout Claw Protect documentation, code, and API surfaces.

| Term | Definition |
|------|-----------|
| **Agent** | An autonomous AI process (e.g., OpenClaw, Hermes) that performs tasks on behalf of a user. Agents connect to the Claw Protect control plane via the SDK and report telemetry. |
| **Agent Fleet** | The collection of all agents registered to a single tenant. Fleet-level metrics are available in the Analytics dashboard. |
| **Air-Gap** | A network configuration where agents have no direct outbound internet access. Claw Protect supports air-gapped deployments via the Gateway Relay. |
| **Alert** | A structured notification generated by the Detection Engine when a security rule fires. Alerts have a severity (low / medium / high / critical) and a type (e.g., `prompt_injection`, `data_exfiltration`). |
| **Approval Request** | A request from an agent to perform a sensitive action. The `approvalRequestValidator` module evaluates the request against policy before granting access. |
| **Beaconing** | A data-exfiltration technique where an agent sends small, regularly spaced outbound transfers to signal a command-and-control server. Detected by `dataExfiltrationMonitor`. |
| **Behavioural Baseline** | A statistical profile of an agent's normal activity patterns (action types, frequency, resource access). Used by `agentMonitor` to detect drift. |
| **Behavioural Drift** | When an agent's activity deviates significantly from its established baseline, possibly indicating compromise or misconfiguration. |
| **Control Plane** | The central Claw Protect service that receives telemetry, runs detection modules, stores data, and serves the dashboard. Consists of the API Gateway, Detection Engine, Governance Engine, and Dashboard. |
| **Credential Rotation** | The process of periodically replacing secrets (JWT keys, API keys, signing keys) to limit the blast radius of a compromise. See ARCHITECTURE.md §4.2 for cadences. |
| **CMEK** | Customer-Managed Encryption Keys. Enterprise-tier feature allowing tenants to control their own encryption keys for Firestore data at rest. |
| **CVE** | Common Vulnerabilities and Exposures. A publicly disclosed security flaw. Claw Protect's CVE management process is described in ARCHITECTURE.md §5.4. |
| **Data Residency** | The geographic region where a tenant's data is stored. Claw Protect supports US, EU, APAC, and Canada regions (see ARCHITECTURE.md §7). |
| **Detection Engine** | The control plane component that runs all 20 security modules in parallel against incoming telemetry to produce alerts and risk scores. |
| **DLP (Data Loss Prevention)** | Telemetry and controls related to preventing unauthorized data disclosure. Includes secret scanning, PII detection, and outbound transfer monitoring. |
| **Exfiltration** | Unauthorized transfer of data from an agent or the control plane to an external destination. |
| **Gateway Relay** | An optional component deployed inside restricted networks that terminates agent connections locally, buffers telemetry, and forwards it to the control plane over a single outbound TLS connection. |
| **Governance Engine** | The control plane component responsible for compliance policy evaluation, audit log management, and credential rotation scheduling. |
| **Harvest Attack** | An attack where an adversary captures encrypted traffic now with the intent to decrypt it later using a future quantum computer. Mitigated by post-quantum encryption. |
| **ITDR** | Identity Threat Detection and Response. A zero-trust capability in `zeroTrustManager` that detects identity-based attacks (credential stuffing, token replay, privilege escalation). |
| **Least Privilege** | The security principle that agents should only have the minimum permissions required for their tasks. Enforced by `permissionAnalyzer`. |
| **Module** | A self-contained security capability within Claw Protect (e.g., `promptInjectionDetector`, `zeroTrustManager`). Each module exports typed interfaces and operates as a singleton. |
| **mTLS** | Mutual TLS. A protocol where both client and server present certificates for authentication. Used between Gateway Relay and agents. |
| **Offboarding** | The process of decommissioning an agent: revoking credentials, terminating sessions, and marking historical data as decommissioned. |
| **Onboarding** | The process of registering a new agent with the control plane: issuing a registration token, establishing a session, and setting an initial trust score. |
| **OpenClaw** | An open-source AI agent framework with multi-agent orchestration. Claw Protect is an enterprise-hardened, OpenClaw-inspired security layer—it builds on OpenClaw patterns but is independently developed and maintained. |
| **Payload** | A JSON object exchanged between agents and the control plane. Three payload types exist: `telemetry`, `alert`, and `config` (see `schemas/claw-protect-payload.schema.json`). |
| **Playbook** | An automated incident response workflow defined in `playbookEngine`. Playbooks consist of steps (conditions, actions, approvals) that execute in sequence or parallel. |
| **Post-Quantum Cryptography (PQC)** | Cryptographic algorithms resistant to attacks by quantum computers. Claw Protect supports NIST-approved algorithms: ML-KEM (key encapsulation), ML-DSA (digital signatures), SLH-DSA (hash-based signatures). |
| **Pricing Tier** | The subscription level (`free`, `starter`, `pro`, `enterprise`) that determines rate limits, data retention, feature access, and data residency options. |
| **Prompt Injection** | An attack where malicious instructions are inserted into an agent's input to manipulate its behaviour. Includes direct injection, indirect (web) injection, and delimiter-based attacks. |
| **Pseudonymisation** | Replacing identifying information with artificial identifiers. Agent IDs and user IDs in Claw Protect are pseudonymous by default. |
| **Runaway Loop** | A condition where an agent repeats the same actions excessively (default threshold: 50+ actions in 5 minutes), often indicating a bug or attack. |
| **SBOM** | Software Bill of Materials. A machine-readable inventory of all components in a software system. Generated by `agentDependencyGraph`. |
| **Session** | A time-bounded connection between an agent and the control plane. Sessions have a dynamic trust score (0–100) that adjusts based on behaviour. |
| **Shadow Agent** | An unregistered or unauthorized agent operating within a network. Discovered by `shadowAgentDiscovery` through network, tool usage, and API activity scanning. |
| **Skill** | A capability or tool that an agent can load and execute at runtime. Skills are cryptographically signed and verified before loading (see ARCHITECTURE.md §4.3). |
| **Spoofing** | An attack where one agent impersonates another. Detected by `agentIdentityManager` through challenge-response authentication. |
| **Telemetry** | Structured security signals sent from agents to the control plane at regular intervals. Includes endpoint, network, DLP, and identity data. |
| **Threat Score** | A normalized 0–100 metric representing the current security risk level. Composed from multiple module outputs. |
| **Triple Extortion** | A ransomware attack that combines data encryption, data theft, and threats against the victim's partners or customers. Detected by `ransomwareDefense`. |
| **Trust Score** | A dynamic 0–100 metric assigned to each agent session by `zeroTrustManager`. Sessions below the configured minimum are denied access. |
| **Typosquatting** | An attack where a malicious tool or package uses a name similar to a legitimate one (e.g., `langchian` vs. `langchain`). Detected by `toolSupplyChainVerifier` using Levenshtein distance. |
| **Zero Trust** | A security model that requires verification for every access request regardless of network location. Claw Protect implements zero trust at session, module, and network levels. |
